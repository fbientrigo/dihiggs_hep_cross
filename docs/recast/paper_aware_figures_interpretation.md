# Interpreting the paper-aware figures

Generated by `scripts/04_make_paper_aware_figures.py` into `outputs/figures/`.
**None of these six figures is an ATLAS reproduction, an ATLAS-validated
efficiency, or an exclusion.** Read `docs/recast/current_limitations.md`
first if you haven't.

## Figure 1 — ID decay probability sweet spot

Pure analytic geometry (`decay_probability_between_radii` from
`recast_math.py`), no HEPData input. Shows `P(4 mm < R < 300 mm)` vs `ctau`
for four assumed `beta*gamma` values.

**What it shows:** the DV search only "sees" a narrow lifetime window. A
particle that decays too early (before 4 mm) looks prompt and is vetoed by
upstream selections; one that decays too late (past 300 mm) escapes the
inner-detector-based reconstruction this analysis relies on.

**What it does not show:** a real `beta*gamma` distribution. Each curve
assumes every S in every event has exactly that one boost — real events have
a spread set by the `gg -> h* -> SS` kinematics (mS, sqrt(s), parton
luminosities), which we don't have without signal MC.

## Figure 2 — Trackless event efficiency vs Sumpt

Input: `outputs/hepdata_tidy/event_efficiency_trackless.csv`, three radial
categories encoded only in the source filename.

**What it shows:** ATLAS's published event-level trigger/selection
efficiency is not flat — it depends on both the summed track pT in the event
and which radial shell the vertex falls in. This is a genuine, nontrivial
piece of detector-level physics that public HEPData products encode and that
a recast can reuse directly.

**What it does not show:** scalar-S efficiency. This is the ATLAS SUSY EWK
(chargino/neutralino) benchmark's own efficiency curve, published for a
different production mode and different mass parameter (neutralino mass, not
mS). See `EWK_BENCHMARK_NOT_SCALAR_S` on this figure and in
`docs/recast/current_limitations.md`. Track/jet activity from
`h* -> S S -> hadrons` may populate Sumpt very differently than
chargino/neutralino decay products.

Note the `R < 1150 mm` curve peaks above 1.0 (up to ~1.19): this is not a
bounded efficiency in the usual sense, it is HEPData's own published
`value` for this observable, copied as-is -- not a plotting bug or a
normalization error on our side.

## Figure 3 — Vertex efficiency heatmaps

Input: `outputs/hepdata_tidy/vertex_efficiency_grid.csv`, three representative
radial bins (22-25 mm, 84-111 mm, 180-300 mm) out of 12 published.

**What it shows:** decaying inside the fiducial radius is necessary but not
sufficient. Vertex mass (`m_DV`) and track multiplicity (`n_tracks`) gate
efficiency further, and the shape of that gate changes with radius — the
22-25 mm bin reaches efficiencies near 0.5 for high track count/mass, while
the 180-300 mm bin stays low everywhere in the sampled range. This is exactly
the kind of structure a flat placeholder efficiency (as currently used in
`scripts/03_paper_s_benchmark_proxy.py`) cannot capture.

**What it does not show:** scalar-S vertex kinematics. `m_DV` and
`n_tracks` here are for chargino/neutralino decay products, not `S -> qq/gg`
decay products. Until truth-level Pythia kinematics exist for the scalar S,
we cannot look up a physically appropriate cell in these grids.

## Figure 4 — Trackless cutflow

Input: `outputs/hepdata_tidy/cutflow_trackless_ewk.csv`, all 4 published
mass/lifetime working points (500 GeV/0.1 ns, 500 GeV/1.0 ns, 1300 GeV/0.1 ns,
1300 GeV/1.0 ns), log-scale `A x epsilon` per selection step.

**What it shows:** where signal is progressively lost through the trigger ->
filter -> jet selection -> DV geometry -> material veto -> track-count ->
DV-mass -> selected-tracks chain. Note the two lifetimes (0.1 ns vs 1.0 ns)
track each other closely through jet selection, then diverge specifically at
the material-veto/track-count steps — lifetime interacts with the selection
logic, not just with production or geometric acceptance.

**What it does not show:** a scalar-S cutflow. These are ATLAS's own
benchmark working points; the absolute `A x epsilon` values are specific to
chargino/neutralino production and decay-product multiplicity. The *shape*
of per-step loss is the transferable lesson, not the numbers themselves.

## Figure 5 — HEPData inventory summary

Input: `outputs/hepdata_inventory/atlas_dvjets_table_inventory.csv`.

**What it shows:** the public HEPData bundle (96 tables) is not just
headline exclusion plots — it contains real recast ingredients: 12
vertex-efficiency grids, 6 event-efficiency tables, 4 cutflows, 4 acceptance
tables, 6 yield tables, 3 cross-section-limit tables. That structural
richness is *why* a recast scaffold is possible at all from public data
alone, without rerunning ATLAS's detector simulation.

**What it does not show:** relevance to scalar-S. Table *count* says
nothing about table *applicability* — the largest category
(`exclusion_limits`, 60 tables) is EWK-benchmark-specific and not directly
reusable for a different production/decay model.

## Figure 6 — Scalar-S proxy map

Input: `outputs/paper_s_proxy/paper_s_benchmark_proxy.csv` (3 masses x 7
lifetimes, dense grid), `Nsig_139fb` proxy value per cell.

**"PAPER-AWARE PROXY -- NOT EXCLUSION GRADE"** is printed directly on the
figure because this is the panel most likely to be misread as a result. It
shows where in `(mS, ctau)` space the *current*, placeholder-heavy pipeline
predicts the most proxy signal — useful for prioritizing which benchmark
point to validate first with real MC, and nothing more.

**What it does not show:** an exclusion, a sensitivity estimate, or anything
comparable to the paper's own figures. `sigma_fb_assumed` is a flat 1 fb
placeholder, `beta_gamma_assumed` is a flat 1.0, `BR_hadronic_proxy` is a
step function, and `eff_vertex_proxy` is borrowed from the EWK cutflow's
final step. Every one of these is listed per-row in the proxy CSV's
`missing_inputs` column.

## Summary: what would upgrade these figures

All six figures share the same missing ingredient identified in
`docs/recast/paper_2606_01681_operational_notes.md` section 3: real
FeynRules/UFO -> MadGraph -> Pythia signal MC for
`pp -> gg -> h* -> S S`, giving a true `beta_gamma` distribution, a true
`sigma_fb`, and truth-level DV kinematics (`m_DV`, `n_tracks`, `Sumpt`) that
can be looked up directly in the Figure 2/3 tables instead of averaged or
assumed.
