# Minimum diphoton recast contract

Three tables, joined on mass (+ width hypothesis). Generated by
`scripts/08_diphoton_meeting_package.py` into
`outputs/diphoton_recast_contract/{theory_side,experiment_side,comparison}.csv`.

## Theory side (from 2HDMC scan, not yet wired up)

```text
point_id, m_H_GeV, Gamma_H_GeV, Gamma_over_m, br_gaga,
sigma_ggF_fb, sigma_VBF_fb, sigma_total_fb, sigma_times_br_gaga_fb,
production_mode, validity_flags
```

Mapping from our 2HDMC scan columns:

```text
m_H_GeV        <- m_phi
Gamma_H_GeV    <- total_width
Gamma_over_m   <- total_width / m_phi
br_gaga        <- br_gaga
sigma_ggF_fb   <- NOT YET AVAILABLE (needs external xsec tool)
sigma_VBF_fb   <- NOT YET AVAILABLE (needs external xsec tool)
sigma_times_br_gaga_fb <- sigma_total_fb * br_gaga (only once sigma exists)
```

Today's CSV holds 2 `ILLUSTRATIVE_*` rows (masses only) — placeholders
that exercise the join logic, not real 2HDMC output. No cross section is
invented anywhere; every xsec/BR field a real scan hasn't filled in is
`None`.

## Experiment side (from ATLAS HIGG-2018-27, real data)

```text
mass_GeV, width_hypothesis, Gamma_over_m,
observed_limit_fb, expected_limit_fb,
expected_minus_1sigma_fb, expected_plus_1sigma_fb,
expected_minus_2sigma_fb, expected_plus_2sigma_fb,
fiducial_or_total, spin_assumption, source_table
```

727 real rows: 4 width hypotheses (NWA/2%/6%/10%) x their respective mass
grids (160-3000 GeV for NWA, 400-2800 GeV for the others).
`fiducial_or_total = "fiducial"` for all rows (see dataset card
limitations). Source: `Limit1D_{NW,LW002,LW006,LW01}_Scalar.yaml`.

## Comparison

```text
point_id, nearest_mass_GeV, nearest_width_hypothesis,
theory_over_observed_limit, theory_over_expected_limit,
excluded_observed_proxy, excluded_expected_proxy, quality_flag
```

`theory_over_*_limit` and `excluded_*_proxy` are `None`/`UNKNOWN_NO_XSEC`
for every row today, because `sigma_times_br_gaga_fb` doesn't exist yet.
This is intentional, not a bug: the contract table is complete and ready
to populate the instant a real cross section is supplied.

## Quality flags (defined, applied in the JSON/CSV outputs)

```text
NOT_EXCLUSION_UNTIL_XSEC_SUPPLIED   — no theory sigma yet, comparison is undefined
WIDTH_MATCH_APPROXIMATE             — nearest of 4 discrete width hypotheses used
MASS_INTERPOLATION_APPROXIMATE      — nearest mass grid point used, not interpolated
FIDUCIAL_VS_TOTAL_CHECK_REQUIRED    — limit is fiducial; C_X*A_X not yet applied
SPIN0_ASSUMPTION_OK_FOR_2HDM        — a 2HDM H2 is spin-0, matches the search's
                                       spin-0 hypothesis (not independently verified
                                       against production-mode-specific acceptance)
```

## What's still missing before any real comparison

1. A 2HDM scan CSV with `m_phi`, `total_width`, `br_gaga` for real
   points (the columns exist per the task brief; no such file lives in
   this repo yet). The moment one exists, drop it in `data/2hdmc_scans/`
   and follow [`2hdmc_handoff_runbook.md`](2hdmc_handoff_runbook.md).
2. A cross-section tool (e.g. SusHi/HIGLU-style ggF, or VBF) to turn a
   2HDM point into `sigma_ggF_fb`/`sigma_VBF_fb`. Not run this session —
   out of the 30-minute compute budget and out of scope ("do not invent
   cross sections").
3. Once both exist: `sigma_times_br_gaga_fb = sigma_total_fb * br_gaga`,
   join to nearest `(mass_GeV, width_hypothesis)` in the experiment
   table, compute the two ratio columns, and only then can
   `excluded_*_proxy` become a real yes/no (still a *proxy*, not a
   validated CLs computation).
